# 动态规划系统使用指南

## 系统概述

研途智伴的动态规划系统是一个真正区别于普通聊天模型的核心功能。它不仅能回答"怎么学"，还能制定"具体每天学什么"，并根据实际进度动态调整。

### 核心特性

1. **动态规划**：根据学习进度自动调整计划
2. **状态管理**：记录和跟踪每日学习进度
3. **闭环反馈**：规划→执行→检查→调整的完整循环
4. **智能顺延**：未完成任务自动加入后续计划

## 工作流程

### 早上流程（7:30）

```
步骤1：读取周计划
  └─ 获取本周整体目标
  └─ 查看各科任务清单
  └─ 检查进度追踪表

步骤2：读取昨日进度
  └─ 了解昨天实际完成情况
  └─ 查看学习时长和状态
  └─ 获取"明日建议"

步骤3：制定今日计划
  原则：
  - 绝不私自添加或修改任务
  - 所有任务必须来自周计划
  - 根据昨天完成情况动态调整
  
  逻辑：
  - 昨天完成好（≥80%）→ 按原计划推进
  - 昨天完成一般（50-80%）→ 减少任务量
  - 昨天完成差（<50%）→ 先补昨天，再考虑新任务

步骤4：发送计划预览
  └─ 显示今日任务列表
  └─ 显示计划学习时长
  └─ 按科目分类显示
```

### 晚上流程（22:00）

```
步骤1：提醒上报
  └─ 提醒学生上报今日学习时长
  └─ 询问计划完成情况

步骤2：记录进度
  └─ 创建进度记录文件
  └─ 记录完成情况、学习时长、状态评价

步骤3：调整明日计划
  └─ 根据今日完成度动态调整
  └─ 生成明日建议
  └─ 预览明日计划调整
```

### 周总结流程（周日）

```
步骤1：汇总周进度
  └─ 统计各日完成情况
  └─ 计算周平均完成度
  └─ 分析各科目完成情况

步骤2：分析完成情况
  └─ 对比计划vs实际
  └─ 识别薄弱环节
  └─ 评估学习效率

步骤3：制定下周方案
  └─ 生成改进建议
  └─ 调整下周计划
  └─ 设定新的目标
```

## 动态调整逻辑

### 调整规则

| 昨日完成度 | 调整策略 | 说明 |
|----------|--------|------|
| ≥80% | 按原计划推进 | 完成度好，保持节奏 |
| 50-80% | 减少任务量（80%） | 完成度一般，适度减少 |
| <50% | 先补昨天，减少新任务（50%） | 完成度差，优先补缺 |

### 任务顺延

- 未完成的任务自动标记为"顺延"
- 顺延任务优先级提升
- 记录顺延来源和日期
- 自动加入后续计划

## 使用示例

### 场景1：完成度好的情况

```
昨天完成度：90%
今天原计划：8小时，4个任务

调整结果：
✅ 按原计划推进
- 任务1：泰勒公式复习（2小时）
- 任务2：做数学真题（2小时）
- 任务3：英语阅读理解（2小时）
- 任务4：政治知识点总结（2小时）

建议：
💡 今日学习效果优秀，保持当前学习节奏
💡 可以考虑增加难度或深度学习
```

### 场景2：完成度一般的情况

```
昨天完成度：65%
今天原计划：8小时，4个任务

调整结果：
⚠️ 减少任务量（80%）
- 任务1：泰勒公式复习（2小时）
- 任务2：做数学真题（1.6小时）
- 任务3：英语阅读理解（1.6小时）
- 任务4：政治知识点总结（1.6小时）

建议：
📝 建议明天保持相同的任务量
🎯 重点关注未完成的任务，优先完成
```

### 场景3：完成度差的情况

```
昨天完成度：35%
昨天未完成任务：
- 任务1：泰勒公式复习（未完成）
- 任务2：做数学真题（未完成）

今天原计划：8小时，4个任务

调整结果：
❌ 先补昨天，减少新任务（50%）
- [顺延] 任务1：泰勒公式复习（2小时）
- [顺延] 任务2：做数学真题（2小时）
- 任务3：英语阅读理解（1小时）
- 任务4：政治知识点总结（1小时）

建议：
❌ 今日完成度较低，建议明天大幅减少任务量
🔍 分析未完成原因，调整学习策略
⏰ 检查是否有时间管理问题
```

## 常见问题

### Q: 系统如何区别于普通聊天模型？

A: 
- **普通模型**：只能回答"怎么学"（一次性建议）
- **我们的系统**：能制定"具体每天学什么"（持续规划）
- **关键区别**：
  1. 状态管理：记录每日进度
  2. 动态调整：根据实际情况调整计划
  3. 闭环反馈：规划→执行→检查→调整
  4. 智能顺延：未完成任务自动处理

### Q: 如何处理计划外的情况？

A: 系统会在晚上流程中记录问题，并在次日早上根据实际情况调整计划。

### Q: 如何查看学习效率？

A: 调用 `/api/ai/plan/statistics/efficiency` 接口，系统会计算实际学习时长与计划时长的比率。

### Q: 如何重新制定计划？

A: 调用 `/api/ai/plan/weekly/init` 接口初始化新的周计划。
