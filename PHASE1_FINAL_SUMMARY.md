## 🎯 核心功能实现

### 1. 周计划创建（自然语言输入）

**用户输入**：
```
"我这周的考研计划是：数学要把张宇30讲第一讲搞透彻，完成660函数极限连续，
完成1000题第一章基础题，每天控制在5-6h以内。英语一天200英语单词、看完颉斌斌语法。
408要把1.1-3.2看完，一天最多4h，只做选择题。"
```

**系统处理**：
1. 接收自然语言描述
2. 调用AI整理和结构化
3. 生成markdown格式的周计划
4. 自动保存到文件系统

**输出**：
```
第一周（2026-03-10-2026-03-16）周计划.md
├── 本周核心目标
├── 各科复习任务
├── 周进度追踪表
├── 每日学习时间分配
└── 学习建议
```

### 2. 早上流程（自动生成今日计划）

**自动执行**：
```
早上7:30 → 读取周计划 + 读取昨日进度
         → 分析昨日完成度
         → 根据完成度动态调整
         → 生成今日计划
```

**动态调整规则**：
```
昨天完成度 > 80%  → ✅ 按原计划推进
昨天完成度 50-80% → ⚠️ 减少20%任务量
昨天完成度 < 50%  → ❌ 重复昨天未完成内容
```

**输出示例**：
```
🌅 早上好！我检查了周计划和昨天进度，这是今天的计划预览

📅 今日学习计划（2026-03-10）
================

📐 数学（5.5小时）
- 学习张宇30讲第一讲「函数极限连续」前半部分
- 完成660题「函数极限连续」模块15道选择题

📝 英语（2小时）
- 背诵200个考研核心词汇
- 看1节颉斌斌语法课程

💻 408数据结构（3.5小时）
- 学习数据结构1.1-1.5章节
- 完成对应章节选择题
```

### 3. 晚上流程（自动记录进度）

**用户输入**（自然语言）：
```
"学了6小时，数学完成了80%，英语完成了100%，408完成了70%，状态一般，数学有些难"
```

**系统处理**：
1. 解析自然语言进度
2. 提取各科完成度、学习时长、状态等
3. 生成markdown格式的进度记录
4. 自动保存到文件系统
5. 生成明日建议

**输出**：
```
2026-03-10_学习进度记录.md
├── 📊 今日完成情况
│   ├── 📐 数学（80%）
│   ├── 📝 英语（100%）
│   └── 💻 专业课（70%）
├── 📈 今日状态评价
│   ├── 学习时长：6.0小时
│   ├── 总体完成度：83%
│   ├── 状态评价：一般
│   └── 遇到的问题：数学有些难
└── 🎯 明日建议
    ├── ✅ 今日完成度优秀（83%），明日按原计划推进
    ├── 💡 可以考虑增加难度或深度学习
    └── 👍 保持当前学习节奏和方法
```

### 4. 闭环反馈系统

```
Day 1:
  早上 → 生成计划（第一天无昨日进度，按原计划）
  晚上 → 上报进度（完成度83%）
         ↓
Day 2:
  早上 → 读取昨天进度（83%>80%）
       → 按原计划推进
       → 生成今日计划
  晚上 → 上报进度（完成度82%）
         ↓
Day 3:
  早上 → 读取昨天进度（82%>80%）
       → 按原计划推进
       → 生成今日计划
  ...循环
```

## 📡 API 接口（6个）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/ai/plan/v2/init` | POST | 初始化工作目录 |
| `/ai/plan/v2/weekly/create` | POST | 创建周计划（自然语言输入） |
| `/ai/plan/v2/daily/morning` | GET | 早上流程（生成今日计划） |
| `/ai/plan/v2/daily/evening` | POST | 晚上流程（上报学习进度） |
| `/ai/plan/v2/weekly/read` | GET | 读取周计划 |
| `/ai/plan/v2/progress/read` | GET | 读取进度记录 |


## 💡 与普通聊天模型的区别

### 普通聊天模型
```
用户：我怎么学考研？
模型：建议1、建议2、建议3...
（一次性回答，无后续跟踪）
```

### 我们的系统
```
用户：输入周计划描述
系统：生成结构化周计划

每天早上：自动生成今日计划
每天晚上：自动记录进度
明天早上：根据昨天进度调整今天计划

持续循环，不断优化
```
