# Phase 1 完整使用指南 - 包含周计划创建

## 🎯 完整的工作流程

```
第一步：初始化工作目录
  ↓
第二步：创建周计划（用户输入自然语言 → AI整理成markdown）
  ↓
第三步：每天早上生成今日计划（自动读取周计划+昨日进度）
  ↓
第四步：每天晚上上报学习进度（自然语言输入 → AI解析 → 生成记录）
  ↓
循环...
```

### 2️⃣ 创建周计划（核心功能）

**请求**：
```bash
POST http://localhost:8123/ai/plan/v2/weekly/create
Content-Type: application/json

{
  "weekDescription": "我这周的考研计划是：数学要把张宇30讲第一讲搞透彻，完成660函数极限连续，完成1000题第一章基础题，每天控制在5-6h以内。英语一天200英语单词、看完颉斌斌语法。408要把1.1-3.2看完，一天最多4h，只做选择题。",
  "startDate": "2026-03-10",
  "endDate": "2026-03-16"
}
```

**响应**：
```json
{
  "success": true,
  "message": "✅ 周计划已创建并保存",
  "filename": "第一周（2026-03-10-2026-03-16）周计划.md",
  "weeklyPlan": "# 第一周（2026.3.10-2026.3.16）周计划\n\n## 本周核心目标\n- 数学：完成张宇30讲第一讲，掌握函数极限连续\n- 英语：背诵1400个单词，看完颉斌斌语法\n- 408：完成数据结构1.1-3.2章节\n- 总学习时间：50小时\n\n## 各科复习任务\n\n### 📐 数学（每日5-6小时）\n1. 学习张宇30讲第一讲「函数极限连续」\n2. 完成660题「函数极限连续」模块\n3. 完成1000题第一章基础题\n\n### 📝 英语（每日2小时）\n1. 背诵200个考研核心词汇\n2. 看颉斌斌语法课程\n\n### 💻 408数据结构（每日4小时）\n1. 学习1.1-3.2章节内容\n2. 完成对应章节选择题\n\n## 周进度追踪表\n| 科目 | 任务 | 完成情况 |\n|------|------|----------|\n| 数学 | 张宇30讲第一讲 | ⬜️ 未完成 |\n| 英语 | 颉斌斌语法 | ⬜️ 未完成 |\n| 408 | 数据结构1.1-3.2 | ⬜️ 未完成 |\n\n## 每日学习时间分配\n- 总学习时间：11小时/天\n- 数学：5.5小时/天\n- 英语：2小时/天\n- 408：3.5小时/天\n\n## 学习建议\n1. 数学是重点，每天保证5-6小时\n2. 英语要坚持每天背单词\n3. 408选择题为主，不做大题"
}
```

**说明**：
- 系统会自动调用AI整理用户的自然语言描述
- 生成结构化的markdown文件
- 自动保存到 `./workspace/周计划/` 目录

### 3️⃣ 早上流程 - 生成今日计划

**请求**：
```bash
GET http://localhost:8123/ai/plan/v2/daily/morning?date=2026-03-10
```

**响应**：
```json
{
  "success": true,
  "date": "2026-03-10",
  "message": "🌅 早上好！我检查了周计划和昨天进度，这是今天的计划预览\n\n📅 今日学习计划（2026-03-10）\n================\n\n📐 数学（5.5小时）\n- 学习张宇30讲第一讲「函数极限连续」前半部分\n- 完成660题「函数极限连续」模块15道选择题\n- 巩固基础公式\n\n📝 英语（2小时）\n- 背诵200个考研核心词汇\n- 看1节颉斌斌语法课程\n\n💻 408数据结构（3.5小时）\n- 学习数据结构1.1-1.5章节\n- 完成对应章节选择题\n\n💡 学习建议\n- 上午：完成数学任务（效率最高）\n- 下午：英语阅读理解\n- 晚上：408学习 + 单词背诵"
}
```

**说明**：
- 系统自动读取周计划文件
- 如果有昨日进度，会根据完成度动态调整
- 第一天没有昨日进度，按原计划推进

## 🔑 核心特性

### 1. 自然语言周计划输入
```
用户输入：
"我这周的考研计划是：数学要把张宇30讲第一讲搞透彻，完成660函数极限连续..."

AI输出：
结构化的markdown周计划文件
```

### 2. 自动读取和分析
```
早上：自动读取周计划 + 昨日进度
      → 分析完成度
      → 动态调整
      → 生成今日计划
```

### 3. 动态调整规则
```
昨天完成度 > 80%  → 按原计划推进
昨天完成度 50-80% → 减少20%任务量
昨天完成度 < 50%  → 重复昨天未完成内容
```

### 4. 自然语言进度输入
```
用户输入：
"学了6小时，数学完成了80%，英语完成了100%，408完成了70%，状态一般，数学有些难"

AI输出：
- 解析各科完成度
- 生成进度记录文件
- 生成明日建议
```
